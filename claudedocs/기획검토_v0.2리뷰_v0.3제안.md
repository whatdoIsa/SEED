# 주린이 앱 기획서 v0.2 검토 및 v0.3 디벨롭 제안

> 검토 기준: 제품 전략 · 학습 설계 · 솔로 개발 실행 가능성 · 디자인 시스템
> 작성일: 2026-07-06

---

## 1. 총평

v0.2는 방향이 잘 잡힌 기획서다. 특히 세 가지가 강하다.

1. **§9.1 예측 출력 가드레일** — 점 가격 예측 금지, 순위·확률 표현 원칙은 법적 방어이자 교육 철학으로 일관성이 있다. 경쟁 앱 대부분이 놓치는 지점이다.
2. **수익화 3층 구조의 논리** — "넓이가 아니라 깊이를 게이팅한다", IAP(고정 콘텐츠)와 구독(반복 가치)의 분리 근거가 명확하다.
3. **차별화 wedge 정의** — 살아있는 엔진 + 실패 시나리오 + AI 복기의 결합은 실제로 경쟁 앱과 다르다.

그러나 구조적 이슈가 두 개 있다. 이 두 개는 기능 추가가 아니라 **기획의 뼈대 수정**이 필요한 수준이다.

- **P0에 차별화가 하나도 없다.** 4장에서 정의한 "제품의 심장" 세 가지가 전부 P1 이후다. MVP는 기존 모의투자 앱의 하위호환이 된다.
- **시간 압축(배속)이 어디에도 없다.** 커리큘럼(이평선·지지저항·터틀·가치투자)은 수십~수백 캔들의 시간 경과가 필요한데, 시뮬레이션은 분봉 실시간이다. 이 충돌은 §11에서 "거장 봇이 분봉과 충돌"이라는 리스크로 감지만 되어 있고 해법이 없다.

아래에서 이슈별로 진단과 구체적 제안을 정리한다.

---

## 2. 구조적 이슈와 제안 (우선순위순)

### 2.1 P0에 차별화가 없다 — 가장 큰 문제

**진단.** 현재 P0 = 시뮬 마켓 + 시장가 매매 + 포트폴리오 + 레슨 1종 + 리셋. 이 구성은 사용자 눈에는 정확히 "또 하나의 모의투자 + 퀴즈"다. §3에서 스스로 "단순 복제는 의미 없음"이라 진단해놓고, 첫 출시물이 단순 복제 형태가 된다.

더 미묘한 문제: **살아있는 엔진은 사용자가 체감으로 구분하지 못한다.** 호가창이 진짜 에이전트 주문으로 움직이는지 미리 그려진 곡선인지, 입문자는 절대 알 수 없다. 내부적으로 아무리 정교해도 증명 장치가 없으면 차별화가 아니다.

**제안 3가지 (모두 P0로).**

1. **슬리피지 튜토리얼 = 엔진의 존재 증명.** 온보딩 미션에서 의도적으로 큰 주문을 내게 한다 → 호가 잔량을 먹으며 평균단가가 밀린다 → "표시 가격은 52,300원이었는데 왜 52,410원에 샀을까?" → 첫 레슨(호가와 체결)로 연결. 미리 그려진 차트 앱에서는 **원리적으로 불가능한 경험**이므로, 이 5분이 "이 앱은 다르다"를 증명한다. 구현 비용도 낮다 — 엔진이 이미 하는 일을 보여주기만 하면 된다.
2. **실패 시나리오 1종을 P0로.** "급등주 추격매수의 결말" 하나만. 시나리오 프리셋은 기술적으로 fairValue 경로 + 에이전트 파라미터 주입이라 P1 항목 중 구현 비용이 가장 낮은 축이다. 첫 출시 스토어 스크린샷과 입소문 문구("일부러 물려보는 앱")를 이것으로 만든다.
3. **라이트 복기(룰베이스)를 P0로.** LLM 없이 태그+결과 데이터 집계만으로 가능(§2.4 참조). AI 복기의 '맛보기'가 아니라 복기 습관 자체를 MVP부터 형성한다.

### 2.2 시간 압축이 미정의 — 커리큘럼과 엔진의 구조적 충돌

**진단.** 레슨 커리큘럼을 시간지평으로 분해하면:

| 레슨 | 필요한 시간 경과 | 분봉 실시간 기준 대기 |
|---|---|---|
| 캔들 읽기 | 캔들 수 개 | 수 분 — OK |
| 이동평균·지지저항 | 20~60 캔들 | 20~60분 — 불가 |
| 터틀(20일 돌파) | 수십~수백 캔들 | 수 시간 — 불가 |
| 가치투자(horizon=.long) | 수백 캔들+ | 원리적으로 불가 |

즉 **커리큘럼의 2번째 레슨부터 이미 실시간 분봉과 충돌한다.** §11의 "거장 봇 분봉 충돌" 리스크의 근본 원인이 이것이고, 아키타입별 시간지평 명시(§12)는 라벨링이지 해법이 아니다.

**제안.**

- **배속 컨트롤을 P0 엔진 요구사항으로 명시.** 1x / 5x / 30x + "다음 캔들로" / "장 마감까지" 스킵. 엔진이 틱 루프 기반이므로 틱 간격 축소로 구현 가능하고, P1의 백그라운드 복귀(경과분 따라잡기)와 동일 메커니즘이라 선행 구현이 오히려 P1 비용을 줄인다.
- **레슨별 시간 스케일 모드 명시.** 호가·체결·스캘핑 감각 레슨 = 분봉 실시간, 이평선·터틀·가치투자 = 압축 모드. 시나리오에서는 "1캔들 = 1일" 프레이밍을 허용한다 — 엔진은 동일하고 라벨만 바꾸면 되며, 교육적으로도 문제없다(오히려 캔들의 스케일 불변성을 가르치는 포인트가 된다).
- 배속 UI는 차트 상단 칩으로(§4.3). 압축 모드 진입 시 "지금은 하루가 1분입니다" 고지.

### 2.3 매매 사유 기록 — 텍스트 입력은 이탈 포인트

**진단.** 매수 버튼과 체결 사이에 자유 텍스트 입력을 끼우는 것은 모바일 UX에서 최악의 마찰 지점이다. 스킵하게 만들면 복기 데이터가 비고, 강제하면 매매 자체가 귀찮아진다. AI 복기의 원료가 되는 데이터인데 수집 설계가 없다.

**제안: 태그 칩 1탭 + 선택적 한 줄.**

- 진입 태그: `돌파 기대` `눌림목` `급등 추격` `뉴스·테마` `그냥 감`
- 청산 태그: `목표 도달` `손절 원칙` `무서워서` `지루해서` `그냥 감`
- 태그 1개는 필수(1탭, 0.5초), 텍스트는 선택.
- 부수 효과가 크다: 태그는 **구조화 데이터**라서 ① 룰베이스 복기가 즉시 가능("`급등 추격` 매수 5건 중 4건 손실, 평균 -3.2%") ② LLM 프롬프트가 짧아져 API 비용 절감 ③ 태그별 성과 집계가 복기 리포트의 핵심 시각화가 된다.
- `그냥 감` 태그를 부끄럽지 않게 디자인할 것 — 솔직한 기록이 데이터 품질의 전제다. 오히려 "감 매매의 성적표"가 가장 강력한 교육 콘텐츠가 된다.

### 2.4 AI 복기 3층 아키텍처 — 비용 문제를 구조로 해결

**진단.** §10에서 "무료 기본 복기(일일/횟수 제한)"라 했지만, 무료 사용자의 복기가 전부 Anthropic API를 태우면 무료층이 곧 비용층이다. "API 비용이 유료화 근거"라는 논리는 맞지만, 무료 맛보기 비용의 상한 설계가 없다.

**제안: 복기를 3층으로 분리.**

| 층 | 엔진 | 비용 | 제공 |
|---|---|---|---|
| L1 통계 복기 | 룰베이스 (태그×결과 집계) | 0원, 오프라인 | 무료 · 무제한 · **P0** |
| L2 요약 복기 | 온디바이스 LLM (Apple Foundation Models) | 0원 | 무료 · 지원 기기 한정 · P1 |
| L3 심층 복기 | Anthropic API | 유료 API | 구독 · P1~P2 |

- L1은 태그 데이터만으로 충분히 강력하다: "당신의 `급등 추격` 매수는 5건 중 4건 손실입니다."
- L2는 iOS 26+/Apple Intelligence 기기에서 L1 통계를 자연어 코멘트로 변환. **주의: 기획서의 iOS 17+ 타깃과 온디바이스 모델 요구사항(iOS 26+)이 다르므로 L2는 가용 기기에서만 활성화, 미지원 기기는 L1 폴백.** 기술 스택 §7에 이 분기를 명시할 것.
- L3만 서버 비용이 발생하고, 여러 세션에 걸친 종단 패턴 분석·철학 렌즈 복기(§12.3) 등 온디바이스로 불가능한 것만 담는다. → "무료 복기 제한"이 인위적 제한이 아니라 **기술 계층의 자연스러운 경계**가 된다.

### 2.5 리셋 설계 — 무제한 리셋은 학습을 파괴한다

**진단.** P0의 "가상 자금 리셋"이 원클릭 무제한이면, 손실의 무게가 사라진다. "가상이니까 몰빵" 행동을 강화하는데, 이는 앱의 학습 목표(리스크 감각)와 정반대다. 모의투자 앱의 고질적 결함이기도 하다 — 여기서 차별화할 수 있다.

**제안.**

- **리셋 전 강제 복기 게이트("계좌 부검").** 리셋 버튼 → 이번 계좌의 사망 원인 리포트(태그별 손익, 최대 실수 1건) → 확인 후 리셋. 리셋 자체를 학습 이벤트로 전환.
- **시즌 프레이밍.** 리셋된 계좌는 "시즌 2, 시즌 3"으로 기록되고, 시즌 간 성적 비교가 성장 그래프가 된다. 리셋 이력을 숨기지 않고 성장 서사로 만든다.
- **파산을 콘텐츠로.** 계좌 -50% 도달 시 특별 레슨 해금: "반토막의 수학 — 원금 회복엔 +100%가 필요하다." 실패 시나리오 철학(§4-2)의 자연 연장.

### 2.6 리텐션 루프 부재

**진단.** §6의 학습 루프는 세션 **내부** 루프다. "내일 왜 다시 열게 되는가"에 대한 답이 기획서에 없다. P2의 "랭킹·챌린지"는 너무 멀고 약하다.

**제안.**

- **오늘의 장 (일일 시나리오 1개).** 매일 다른 시드·상황의 짧은 시나리오. §10.1의 "무료 = 시나리오 일일 제한"과 정확히 맞물린다 — 제한이 아니라 데일리 콘텐츠로 프레이밍.
- **주간 복기 리포트 푸시.** 일요일 저녁 "이번 주 매매 12건, 당신의 패턴이 정리됐어요." 복기 코치가 리텐션 엔진을 겸한다.
- **절제된 스트릭.** §8 톤 원칙(도파민 자극 절제)에 맞게, 불꽃 배지가 아니라 학습 잔디(조용한 캘린더 히트맵) 형태.
- **세션 길이 설계 명시:** 1세션 = 레슨 1 + 미션 1 + 복기 = 5~7분 목표. 출퇴근 시간 사용을 기본 시나리오로.

### 2.7 온보딩 첫 60초

**진단.** 기획서에 온보딩 설계가 없다. 타깃이 "시작이 두려운 사람"인데, 첫 화면에서 차트+호가+체결이 쏟아지면 그 자체가 이탈 사유다.

**제안 플로우.**

1. 가입 없음. 로컬 저장(SwiftData)이므로 가능 — **"회원가입 없이 시작"을 §2 타깃(두려운 사람)에 대한 기획적 응답으로 명시.**
2. "1,000만원이 준비됐어요. 진짜 돈은 1원도 안 듭니다." (모의투자 고지를 안심 메시지로 겸용)
3. Lv0 화면(§3.1): 종목 1개, 가격, 단순 차트, 매수 버튼만.
4. 30초 안에 첫 매수 체결 → 축하가 아니라 관찰 유도: "가격이 계속 움직이죠. 누가 움직이는 걸까요?"
5. 슬리피지 튜토리얼(§2.1) → 첫 레슨.

### 2.8 성공 지표(KPI) — 정의가 없다

MVP가 검증해야 할 가설별로 지표를 미리 정한다.

| 가설 | 지표 | 초기 목표 |
|---|---|---|
| 온보딩이 두려움을 이긴다 | 설치 → 첫 체결 완료율 | 70%+ |
| 학습 루프가 작동한다 | 레슨 1 완주율 / 레슨 2 진입율 | 60% / 40% |
| **복기를 실제로 본다 (핵심 가설)** | 매매 후 복기 리포트 열람률 | 50%+ |
| 다시 온다 | D1 / D7 리텐션 | 35% / 15% |
| 사유 기록이 마찰이 아니다 | 태그 입력 완료율(스킵 불가면 매매 포기율) | 95%+ |

복기 열람률이 가장 중요하다 — 이 숫자가 낮으면 차별화 전략 3번(AI 코치) 전체가 흔들리므로, P1 투자 전에 P0에서 검증한다.

### 2.9 수익화 세부 보완

- **무료 복기 경계 구체화:** L1 무제한 / L2 일 3회 / L3 구독 (§2.4의 계층이 곧 가격 경계).
- **구독 플랜B를 설계 단계에서 준비.** §10.6대로 실데이터 재배포 라이선스가 막히면 구독 기둥의 절반이 무너진다. 라이선스 확인 **전에** 구독 가치를 "실데이터 중심"과 "AI 심층 복기 + 무제한 시나리오 + 월간 챌린지 중심" 두 구성으로 시뮬레이션해두고, 엔타이틀먼트 설계(§10.4)도 데이터 기능을 독립 플래그로 분리해 나중에 빼고 넣기 쉽게 한다.
- **champagne = 유료 시그널 (§3.2 색 시스템 참조).** 브랜드 색을 프리미엄 전용 강조색으로 한정하면 별도 "PRO" 배지 없이 유료 영역이 시각적으로 학습된다.
- 가격 앵커(§10.5)는 유지하되, 검증 방법을 명시: 출시 후 A/B 이전에 ① 스토어 경쟁가 조사(증권플러스·마루랩·해외 Invstr/Bloom 등) ② 베타 테스터 지불의향 인터뷰(Van Westendorp 4문항이면 솔로도 가능).

## 3. 디자인 디벨롭

### 3.0 핵심 결정 — 두 개의 시각 레지스터 (v0.2 리뷰 후 방향 전환)

> 실제 토스증권 화면을 검토한 뒤 디자인 기조를 수정한다. 트레이딩 화면을 bone/champagne 브랜드 톤으로 감싸려던 초안(구 §3.2·§3.3)은 폐기하고, **시장 화면은 토스증권 관례를 충실히 따른다.**

앱을 두 개의 시각 언어로 분리한다.

| 레지스터 | 대상 화면 | 룩앤필 | 근거 |
|---|---|---|---|
| **시장(Market)** | 차트 · 호가 · 주문 · 포트폴리오 | 토스증권 충실 재현 — 흰 배경, 빨강 상승/파랑 하락, 볼드 숫자, 가격 사다리, 체결강도 | **전이 학습.** 여기서 익힌 감각이 실전 토스·키움에 1:1로 옮겨가야 한다. 낯선 UI는 학습을 리셋시킨다. 타깃(두려운 사람)에겐 친숙함이 곧 진입장벽 제거. §8의 TOSS 북극성과 정합. |
| **학습(Learning)** | 온보딩 · 레슨 · 복기 리포트 · 코치 카드 | Arcseed 브랜드 — ink/bone/champagne, 차분한 톤 | 경쟁 앱에 없는 우리만의 영역. 소유 가능한 시각 자산. 시장 화면과 대비되어 "학습이 시장 위에 얹혀 있다"를 시각적으로 전달. |

**전환 자체가 메시지다.** 사용자가 레슨(브랜드 톤)에서 실습(토스 톤)으로 넘어가는 경험이 "이건 실전에서 볼 바로 그 시장이고, 그 위에 학습을 씌운 것"을 말한다.

**차별화는 크롬이 아니라 학습층에서 나온다.** 토스의 `✨ 왜 움직였을까?`(시장 설명)를 우리는 `✨ 왜 이렇게 샀을까?`(내 매매 설명)로 뒤집는다 — 같은 카드 문법, 방향만 반대. 이 한 장이 "토스 복제가 아니라 토스 위의 학습층"임을 증명한다.

**법적 안전선(트레이드 드레스).** 토스를 픽셀 단위로 복제하면 브랜드 혼동 리스크가 있다. 안전선은 **관례는 따르되(빨/파 색규칙·사다리 레이아웃·체결강도 — 업계 표준이라 소유 불가) 폰트·간격·강조색·모의 배지·코치 톤은 우리 것으로** 두는 것. "익숙하되 위조는 아닌" 선을 지킨다. 출시 전 법률 검토(§9) 항목에 트레이드 드레스 1건 추가.

### 3.1 정보 단계 노출 → "도구 해금" 시스템으로 구체화

§8의 "처음엔 단순, 학습 진행 시 상세 해금 고려"를 확정 스펙으로:

| 레벨 | 해금 조건 | 화면에 추가되는 것 |
|---|---|---|
| Lv0 | 첫 실행 | 종목명 · 현재가 · 라인 차트 · 시장가 매수/매도 |
| Lv1 | 캔들 레슨 | 캔들 차트 |
| Lv2 | 거래량 레슨 | 거래량 막대 · 이동평균선 |
| Lv3 | 호가 레슨 | 호가창 (3호가 → 이후 10호가) |
| Lv4 | 체결 레슨 | 체결 테이프 |
| Lv5+ | 지표 레슨 | RSI·MACD 등 오버레이 |

- **해금이 곧 보상이다.** 게임화를 포인트·배지(도파민)가 아니라 "도구를 얻는" 성취로 치환 — §8 톤 원칙과 정합.
- 잠긴 도구는 화면에서 흐릿한 자물쇠 카드로 존재를 보여준다("호가창 · 레슨 3에서 해금"). 다음 레슨의 동기가 된다.
- **Lv-max = 토스 밀도.** 해금의 종착점이 실제 토스증권 수준의 정보 밀도(전체 호가·체결강도·이평선 4종·거래량)다. 즉 "토스보다 단순한 상태에서 시작 → 레슨을 거쳐 토스 밀도까지 도달"이 곧 학습 곡선이다. 실제 토스 호가조차 초보에겐 압도적이라는 사실이 해금 방식의 정당성이다 — 1일차에 전체 호가를 보여주지 않는다.
- 숙련자 배려: 설정에서 "모두 해금" 제공 (잃고 돌아온 두 번째 타깃은 기초 UI 잠금이 모욕적일 수 있음). 온보딩에서 "완전 처음 / 해본 적 있음" 분기 1문항으로 시작 레벨 결정.

| 하락·손실·매도 | `#3182F6` | 토스 관례. 판매하기 버튼 |

| 잔량 틴트 | 매수 `#FEE8EA` / 매도 `#EAF2FE` | 호가창 막대 배경 |
| 이평선 | 5 `#22C55E` · 20 `#F04452` · 60 `#F59E0B` · 120 `#8B5CF6` | 토스와 동일 매핑 |

**학습 레지스터 (Arcseed 브랜드):**

| 역할 | 값 | 용도 |
|---|---|---|
| ink `#16151A` | 텍스트·다크 배경 | 온보딩·레슨·복기 베이스 |
| bone `#F4F1EA` | 라이트 배경 | 학습 화면 캔버스 |

- **역할 분리 명문화:** 시장 화면에서 빨/파는 손익, champagne은 절대 손익 표시에 쓰지 않는다. champagne은 학습층·프리미엄·모의 고지에만.
- 학습 화면은 라이트(bone) 기본 + ink 다크모드 지원. 시장 화면은 토스처럼 흰 배경 기본.
- 크립토 모드(§13.4)의 색 교육 포인트 유지 — 진입 시 1회 카드로 "글로벌 거래소는 상승 초록/하락 빨강, 반대입니다" 고지.

### 3.3 화면별 설계 — 복기 리포트가 시그니처 화면이다

4개 핵심 화면 중 디자인 투자 우선순위를 뒤집을 것을 제안한다. 트레이딩 화면은 관례를 따르면 되지만(오히려 실제 증권앱과 닮아야 전이 학습이 됨), **복기 리포트는 경쟁 앱에 존재하지 않는 화면**이므로 여기가 스토어 스크린샷 1번이자 공유 콘텐츠가 되어야 한다.

**복기 리포트 구성:**
1. 차트 위 내 매매 오버레이 — 매수(B)/매도(S) 마커를 가격 경로 위에 찍어 "내가 어디서 사고 팔았는지"를 한눈에. 꼭지 매수가 시각적으로 자명해지는 순간이 이 앱의 aha 모먼트.
2. 태그별 성과 집계 — `급등 추격` 4건 · 평균 -3.2% (파랑) / `눌림목` 3건 · 평균 +1.8% (빨강).
3. 코치 코멘트 카드 1장 — L1~L3 어느 층이든 형식 통일. 단정 어조 금지, 관찰 어조("~하는 패턴이 보여요").
4. "다음에 시도해볼 것" 딱 1개 — 개선 과제를 쌓아두지 않는다.

복기 리포트는 **학습 레지스터(bone/ink)** 로 그린다 — 시장 화면과 대비되어 "이건 시장이 아니라 너에 대한 화면"임을 톤으로 구분.

**트레이딩 화면(시장 레지스터) 위계:** 상단 `‹ 종목 ♡🔔` → 종목명+모의 배지 → 현재가(최대)+등락 → 탭(차트·호가·내주식·종목정보·커뮤니티) → 차트(이평선 범례·거래량·내주식평균 점선) → 배속/시간대 칩 → 코치 카드 → 하단 고정 `판매하기(파랑)/구매하기(빨강)`. 토스 구조를 그대로 따르되 두 지점만 우리 것으로: ① 배속 칩(시간대 칩 왼쪽, §2.2) ② 코치 카드를 "왜 이렇게 샀을까?"로 뒤집기(§3.0).

**호가 화면(시장 레지스터):** 토스 3열 구조 재현 — 체결 테이프+체결강도 / 가격 사다리(위 빨강·현재가 박스·아래 파랑) / 매도잔량(좌)·매수잔량(우) 막대. 해금 레벨에 따라 노출 호가 수를 조절(Lv3: 3호가 → 이후 10호가). 레슨 진행 중이면 하단에 champagne 힌트 배너로 슬리피지 질문 연결(§2.1).

### 3.4 고지 문구의 디자인 처리

- 상단 상시 배지: `모의투자 · 실제 돈이 아닙니다` — 회색이 아니라 champagne 아웃라인으로, 부끄러운 면책이 아니라 제품 정체성으로.
- 전략·봇 화면 하단 고정: `교육용 전략 · 수익 보장 아님` (§12.3 유지).
- 크립토 유료 진입: `투자 신호가 아닌 학습 데이터입니다` (§13.4 유지).

## 4. 수정된 우선순위 제안 (v0.3 반영용)

### P0 (수정안)
- 살아있는 시뮬 마켓 + 시장가 매매 + 포트폴리오 (기존)
- **시간 배속·스킵 (신규 — §2.2, 엔진 요구사항)**
- **온보딩: 무가입 시작 + 슬리피지 튜토리얼 (신규 — §2.1, §2.7)**
- **실패 시나리오 1종: 급등주 추격매수 (P1에서 승격 — §2.1)**
- **태그 기반 매매 사유 기록 + L1 룰베이스 복기 (신규 — §2.3, §2.4)**
- **도구 해금 Lv0~2 (신규 — §3.1)**
- 레슨 3종: 캔들 / 호가·슬리피지 / 추격매수의 결말 (기존 1종에서 확대)
- 가상 자금 리셋 — **계좌 부검 게이트 포함 (§2.5)**

### P1 (수정안)
- L2 온디바이스 복기 + L3 API 심층 복기 (기존 AI 코치를 계층화)
- 거장 봇 2종 + 터틀 모듈 (기존)
- 실패 시나리오 확장 + 오늘의 장(일일 시나리오) + 주간 리포트 푸시 (§2.6)
- 도구 해금 Lv3~5 (호가창·테이프·지표)
- 백그라운드 복귀 (기존 — 배속과 메커니즘 공유로 비용 절감)
- 결제·엔타이틀먼트 레이어 (수익화는 P1 말)

### 솔로 개발 부하 평가
P0 추가분의 구현 비용: 배속(낮음 — 틱 간격 조절), 시나리오 프리셋 1종(중 — 파라미터 주입), 태그+룰베이스 복기(중 — SwiftData 집계 쿼리), 해금(낮음 — 조건부 노출), 온보딩(중). LLM 복기를 P1로 미룬 것(서버·프롬프트·비용 설계 전부 이연)과 상쇄되어 총량은 크게 늘지 않는다.

### A-1. 트레이딩 · 호가 (시장 레지스터, 토스 충실)
**차트 화면 구성(위→아래):** 상단 `‹ 종목미니 ♡🔔` → `종목명 + 모의 배지(champagne 아웃라인)` → `현재가(최대)+등락(빨/파)` → 탭(차트·호가·내주식·종목정보·커뮤니티, 활성 밑줄) → 이평선 범례(5 `#22C55E`·20 `#F04452`·60 `#F59E0B`·120 `#8B5CF6`) → 캔들+이평선+`내 주식 평균` 점선+우측 현재가 빨강 배지 → **배속 칩(신규)** + 시간대(5분·일·주·월·년) → **코치 카드 `✨ 왜 이렇게 샀을까?`(신규 — 토스 "왜 움직였을까?"의 뒤집기)** → 하단 고정 `판매하기(파랑)/구매하기(빨강)`.
**우리 것 2곳:** ① 배속 칩(시간대 칩 왼쪽, 5배속 등 — §2.2 시간 압축) ② 코치 카드는 시장이 아니라 **내 매매**를 설명(관찰 어조).
**호가 화면:** 토스 3열 — 좌 `체결 테이프+체결강도` / 중 `가격 사다리(위 매도 빨강, 현재가 박스, 아래 매수 파랑, 상·하한가, 시/고 배지)` / 우·좌 `잔량 막대(매수 `#FEE8EA`, 매도 `#EAF2FE`)`. 해금 레벨로 호가 수 조절(Lv3: 4호가 → 이후 10호가). 레슨 중이면 하단 champagne 힌트 배너로 슬리피지 질문 연결.

### A-2. 온보딩 + Lv0 + 해금 경로
**온보딩 환영 [학습]:** ink 로고 → "1,000만원이 준비됐어요 / 진짜 돈은 1원도" → champagne 배지 `회원가입 없이 시작 · 모의투자` → 분기 1문항 `완전 처음이에요 / 해본 적 있어요`(시작 레벨 결정). 무가입 = §2 두려운 타깃에 대한 기획적 응답.
**Lv0 트레이딩 [시장, 벗겨낸 상태]:** 종목명+모의 배지 → 현재가 → **라인 차트(캔들 아님)** → 잠긴 도구 카드 2개(`거래량·이평선 · 레슨2에서 해금`, `호가창·체결 · 레슨3에서 해금`) → 하단 `팔기/사기`. 첫 체결 후 캡션 "가격이 계속 움직여요. 누가 움직이는 걸까요?" → 슬리피지 튜토리얼로 연결.
**해금 경로 [학습]:** Lv0(가격·라인·매매) → Lv1(+캔들) → Lv2(+거래량·이평선) → Lv3(+호가·슬리피지) → **Lv-max(토스 밀도 전체, champagne 강조)**. 잠긴 도구는 화면에 흐릿한 자물쇠 카드로 상시 노출(다음 레슨 동기).

### A-3. 슬리피지 튜토리얼 4단계 (엔진 존재 증명 — 핵심)
감정 곡선: 도발 → 체결 → 충격 → 깨달음.
1. **도발 [시장+ink 코치]:** 최우선 매도호가 150주만 노출 → 코치(ink) "겁내지 말고 크게 질러봐요" + `1,000주 시장가로 사기`(빨강).
2. **체결 [시장]:** 주문이 호가를 아래→위로 먹으며 소진(회색 취소선), 체결 테이프 150@52,300 / 250@52,400 / 300@52,500 / 300@52,550. bone 캡션 "한 가격에 다 못 사요."
3. **충격 [시장]:** `표시가 52,300(취소선) → 평균 체결가 52,460(빨강 대형)`, `슬리피지 +160원(0.31%)` 배지, 체결 내역 표. 코치(ink) "왜 52,300원이 아니었을까요?"
4. **깨달음 [학습]:** 레지스터 전환 → "화면 가격 = 지금 살 수 있는 가장 싼 한 주" → champagne 카드 **차별화 펀치라인** "미리 그려진 차트 앱에선 절대 일어나지 않아요" → `레슨 3 완료` + `다음: 큰 주문 쪼개기`.
**산수 검증:** 150×52,300 + 250×52,400 + 300×52,500 + 300×52,550 = 52,460,000원 ÷ 1,000주 = 52,460원. 표시가 52,300 대비 +160원(+0.31%).
**엔진 스펙 함의:** `MarketEngine`이 시장가 주문의 다단계 체결 내역(가격·수량 배열)과 평균 체결가·슬리피지를 반환해 UI로 노출해야 함. §7 기존 구조 내 작업(신규 아키텍처 아님).

### A-4. 주간 복기 리포트 (학습, 시그니처)
구성: `공유 버튼` → `시즌 N · 주차` → **문장형 헤드라인**("이번 주, 급등을 쫓다 세 번 데였어요") → 지표 카드 3(손익·승률·매매) → **내 매매 지도**(가격 경로 위 B(빨강)/S(파랑) 마커 + "여기서 추격 매수 / 무서워서 손절" 콜아웃 = aha) → **습관 분석**(태그별 건수·평균수익 막대, L1 룰베이스, "손실 대부분이 급등 추격 한 곳") → 코치 카드(ink, 관찰 어조 "사는 속도가 빠를수록 성적이 나빴어요") → **다음 주 딱 한 가지**(단일 과제) → 성장선("시즌 1 대비 급등 추격 40% 감소") → `교육용 복기 · 투자 권유 아님`.
원칙: 숫자보다 문장 먼저 · L1은 오프라인·무료 · 코치는 §9.1 사후 관점 · 개선 과제는 하나만.

### A-5. 계좌 부검 (학습, 리셋 게이트)
리셋 버튼 = 이 화면을 스크롤한 뒤에야 눌리는 학습 게이트.
구성: **ink 헤더 밴드** `시즌 N · 계좌 부검서` + **사망 원인**("자산의 80%를 한 번에") → 계좌 일생(`1,000만 시작 → 최종 485만(-51.5%)` + 자산 곡선: 상승 후 몰빵 지점에서 파랑으로 급락, "최고 1,240만 / 여기서 80% 몰빵" 콜아웃) → **치명적 습관 3**(최대 비중 80% / 급등 추격 12건 중 9건 손실 / 평균 보유 14분) → **반토막의 수학**(ink 카드, 새 레슨 해금 "-51%면 원금까지 +106% 필요") → **시즌 이월 규칙**("돈이 아니라 이것: 한 종목에 30% 이상 넣지 않기") → `부검 확인 · 시즌 N+1 시작하기` + "시즌 기록은 성장 그래프에 남습니다" → 교육용 고지.
원칙: 손실을 정직하게 마주하되 성장 서사로 전환 · 리셋을 이벤트로 · 이월되는 것은 돈이 아니라 규칙.
