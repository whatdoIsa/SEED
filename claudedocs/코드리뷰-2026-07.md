# SEED 전체 코드리뷰 (2026-07-14)

> 4개 영역(엔진·인프라·UI·보안) 병렬 리뷰 후 종합. 모든 발견은 실제 코드를 재확인해 검증한 것만 수록.
> 대상: JurinKit ~4k줄 + SEED 앱 ~15.8k줄 + tutor-worker 96줄 (develop 브랜치)

## 총평

전반적 완성도는 1인 개발 기준 상급이다. 결정론 설계(Dictionary 정렬 소비, 순수함수 기후, Date/전역난수 배제), 원 단위 정수 원장, 소모성 IAP 멱등 지급, 미션 루프의 Task 취소 규율, AI 캐시·폴백 계층, 정직한 페이월 카피는 모두 견고하다.

그러나 심각 결함이 **세 곳에 집중**되어 있다:

1. **iCloud 복원 경로가 사실상 동작하지 않음** — 레슨 진도는 살지만 시장·계좌 연속성은 높은 확률로 파괴되거나 앱이 못 열림 (인프라 C-1+H-1+H-2)
2. **부검 화면의 AI 코치가 매 틱 재생성됨** — Pro 실기기에서만 발현, 시뮬 재현 불가 (UI C-1)
3. **튜터 워커에 비용 폭주 방어가 없음** — deviceId 위조로 쿼터 무한 우회, 실 금전 피해 가능 (보안 H-1)

성능은 복기·내주식 탭이 틱마다 SwiftData 집계/ImageRenderer를 재실행하는 패턴이 반복된다(원인은 전부 동일: `session`의 틱 변동 값을 body에서 직접 관찰).

## P0 — 출시 전 필수 (심각/실피해)

### 1. iCloud 복원 3종 결함 (Critical + High×2)

**C-1. 실행 중 MarketSession이 복원 데이터를 덮어씀**
- `Core/MarketSession.swift:255-263` + `Core/SeedStore.swift:100-114` + `App/RootView.swift:25-31`
- 새 기기 첫 실행: fresh 시드 세션 생성 → CloudKit 임포트 도착 → `refreshAfterRemoteImport()`는 데이터만 갱신하고 세션은 재구성 안 함 → 백그라운드 전환 시 `persistState()`가 임포트된 seed/tick을 fresh 값으로 무조건 클로버 → 다음 실행에서 TradeLog를 **다른 시장**에 리플레이 → 계좌 영구·조용히 왜곡.
- 수정: refresh 후 MarketSession 재구성 트리거.

**H-1. `engineSeedBits`는 어디서도 write되지 않는 죽은 필드 — 시즌 병합이 비결정적**
- `Core/SeedStore.swift:377-386`, `Core/Models.swift:112`
- `active.first { $0.engineSeedBits != nil }`은 항상 nil → 동번호 활성 시즌 2개 중 어느 쪽이 남는지 비결정적, 약 절반 확률로 **임포트된 시즌이 삭제**됨(climateSeed 소실 → 가격 경로 변경).
- 수정: 병합 기준을 SymbolState 보유 여부 또는 lastActiveAt 최신으로 교체.

**H-2. 스냅샷 없는 새 기기에서 30만+ 스텝을 App.init(메인 스레드)에서 동기 리플레이 — 워치독 킬**
- `Core/MarketSession.swift:73-93` + `App/SEEDApp.swift:46-48`
- 폴백 조건이 `>300k && snapshot 존재`인데 LedgerSnapshot은 UserDefaults(기기 로컬)에만 있어 새 기기엔 절대 없음 → 가드 전체가 건너뛰어져 전량 동기 리플레이 → 0x8badf00d로 실행할 때마다 죽는 벽돌 시나리오.
- 수정: 스냅샷 부재 시 리플레이 포기 분기 추가.

> 세 개가 결합하면 광고된 "iCloud 백업·복원"이 시장 연속성에 대해 동작하지 않는다. CloudKit 프로덕션 스키마 배포(보류 중)가 이뤄지기 **전**, 지금이 함께 고칠 적기.

### 2. AutopsyView AI 지문이 매 틱 변경 → 온디바이스 생성 무한 재시작 (Critical)
- `Review/AutopsyView.swift:29-34` + `Learn/AICoachCard.swift:53`
- fingerprint에 `session.totalEquity`를 넣었는데 부검 화면 아래에서 세션 루프가 계속 돌아 매 틱 변동 → `.task(id:)`가 매 틱 취소·재시작 → Pro+Apple Intelligence 기기에서 LanguageModelSession 생성·respond를 초당 반복, AIUsageMeter 부풀림, 캐시 무력화.
- **시뮬에서는 절대 재현 안 됨**(미지원 폴백) — 실기기 검증 전에 반드시 수정.
- 수정: 지문을 시즌 번호/tradeCount 등 정적 값으로, 또는 진입 시점 equity를 @State 스냅샷.

### 3. 튜터 워커 비용 폭주 방어 부재 (High)
- `server/tutor-worker.js:42-54` + `Learn/TutorService.swift:11, 93-99`
- 클라이언트 인증이 전혀 없고 일 30문 상한의 키가 클라이언트 제공 deviceId 그대로 → curl로 요청마다 새 UUID를 넣으면 상한 무의미. 최악의 경우 하루 수백~수천 달러 크레딧 소진 + 정상 사용자 튜터 전면 장애.
- 수정 (우선순위순, 전부 저비용):
  1. Anthropic 콘솔 월 지출 한도 설정 (5분, 최후 방어벽)
  2. 워커에 전역 일일 캡 — KV `global:YYYY-MM-DD` 카운터, 초과 시 429 (~5줄)
  3. 앱-워커 공유 시크릿 헤더 + deviceId UUID 형식 검증
  4. (선택) Cloudflare IP Rate Limiting 규칙 + Bot Fight Mode

### 4. 리필 크레딧이 UserDefaults에만 존재 — 재설치/기기 이전 시 유료 상품 증발 (High, 2개 에이전트 공통 발견)
- `Learn/TutorService.swift:104-125`, `Core/PurchaseStore.swift:104-117, 138-145`
- 소모성 리필(₩1,100/₩2,900)은 지급 즉시 finish되어 구매 복원 불가, 잔액 유일 기록이 UserDefaults → 기기 변경 시 크레딧 0. 환불 요청·별점 1점 직결 유형. "영구 크레딧" 카피와도 불일치.
- 수정: `TutorQuota`·`seed.iap.granted`·`seed.pro.creditMonth`를 `NSUbiquitousKeyValueStore`(iCloud KV)로 이전 — High 중 가장 싼 수정.

## P2 — 출시 후 처리 가능

- **엔진**: stdlib 의존 난수 매핑(H-1 — OS 업데이트 시 전 세션 리플레이 붕괴 가능성, 자체 double01/bounded-int/Fisher-Yates로 선제 고정 권장), 터틀 봇 주문 실패 시 내부 상태 탈동기(`TurtleStrategy.swift:85-117`), 동시호가에 사용자 지정가 미참여(`MarketEngine.swift:278-303`), 부분체결 수수료 청크 반올림의 reservedCash 침식, restore 경로 불변식 검증 부재, `engine.book` 가변 노출
- **인프라**: 지정가 체결 시 스냅샷 미저장(크래시 시 복기-계좌 불일치), 위젯 딥링크 콜드런치 레이스(`RootView.swift:63-68`), 위젯 스트릭 stale + eraseAll 후 위젯 미초기화, `try? context.save()` 전면 침묵(최소 DEBUG 로그), limitFillNotice 타이머 경합
- **UI**: 배너 자동소멸 타이머 경합 3곳(뉴스/미니복기/지정가 — 연속 이벤트 시 조기 소거), QuantBuilder 백테스트 메인 액터 동기 실행(스피너 안 돎), ArenaView ForEach id가 이름(사용자 전략명 충돌 가능), LessonSummarySheet dismiss 직후 fullScreenCover 경합, ETFDetailView NAV 시리즈 매 틱 재구성, "연간 44% 할인" 하드코딩

## 프라이버시·심사

- 튜터 질문 원문이 Cloudflare→Anthropic으로 전송됨 — §1-A 개인정보 조항 게시 + App Privacy 라벨("사용자 콘텐츠, 사용자에 연결 안 됨")은 제출 전 차단 게이트 (기존 할 일과 일치). 튜터 첫 진입 시 한 줄 고지 추가 권장.

- 클라이언트 전용 쿼터/StoreKit 온디바이스 검증은 이 앱 티어에서 수용 가능한 리스크로 판정 (서버 전역 캡 전제).
