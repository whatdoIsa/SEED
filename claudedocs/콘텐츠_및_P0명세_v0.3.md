## Part 1. P0 레슨 3종 스크립트

### 레슨 스키마
각 레슨은 **미니 레슨(읽기) → 미션(시뮬 안 행동) → 즉시 피드백 → 해금**의 짧은 순환이다(§6). 공통 필드:

| 필드 | 내용 |
|---|---|
| 소요 | 목표 시간(읽기+미션) |
| 개념 | 미니 레슨 본문(실제 카피) + 시각 지시 |
| 미션 | 시뮬 안 행동 + 성공 조건 |
| 피드백 | 성공/실패 분기 카피 |
| 해금 | 완료 시 열리는 도구(→ 부록 A-2 해금 경로) |
| 태그 | 이 레슨과 연결되는 매매 태그 |

세 레슨은 사슬이다: **캔들(Lv1 해금) → 호가·슬리피지(Lv3 해금, 슬리피지 튜토리얼 A-3 포함) → 추격매수의 결말(복기 리포트 해금).** 각 레슨이 다음 도구를 연다.

---

### 레슨 1 — 캔들, 시장의 심장박동
- **소요:** ~90초 · **해금:** 캔들 차트(Lv1) · **태그:** 없음(기초)

**개념 (미니 레슨 본문):**
> 방금까지 본 선은 '지금 값' 하나만 보여줬어요. 그런데 5분이든 하루든, 그 시간 동안 가격은 오르락내리락해요. **캔들 하나가 그 시간의 이야기 전부**를 담아요.
>
> 시작보다 **오르며 끝난 시간은 빨강(양봉)**, 내리며 끝난 시간은 **파랑(음봉)**이에요. (외국 앱은 반대예요 — 우리나라는 오르면 빨강이에요.)
>
> 통통한 **몸통**은 시작값과 끝값 사이, 가느다란 **꼬리**는 그 시간의 최고·최저예요. 위로 긴 꼬리는 '올랐다가 도로 밀렸다'는 흔적이에요.

**시각 지시:** 양봉·음봉 각 1개를 크게 띄우고 시가/종가/고가/저가를 라벨로. 위꼬리 긴 캔들 예시 1개.

**미션:** 배속 5x로 다음 캔들 3개가 자라는 걸 지켜본 뒤, **양봉(빨강) 하나를 탭**하세요.

**피드백 분기:**
- 양봉 정답 → "맞아요. 이 시간엔 산 사람들이 이겼어요."
- 음봉 선택 → "이건 파랑, 내리며 끝난 시간이에요. 다시 빨강을 찾아볼까요?"
- 위꼬리 긴 양봉 선택 → "좋아요. 그런데 위꼬리 보이죠? 올랐다가 끝에 밀린 거예요 — 살 땐 이 흔적을 눈여겨봐요."

---

### 레슨 2 — 호가창, 가격 뒤의 진짜 시장
- **소요:** ~2분 · **해금:** 호가창·체결 테이프(Lv3) · **태그:** 이후 매수부터 진입 태그 활성화

**개념 (미니 레슨 본문):**
> 화면의 52,300원은 사실 **'지금 살 수 있는 가장 싼 한 주'**의 값이에요. 그 뒤엔 줄이 서 있어요 — "이 값에 팔게요"(매도호가)와 "이 값에 살게요"(매수호가).
>
> 각 값에 걸린 주식 수를 **잔량**이라고 해요. 최우선 매도호가의 잔량이 적으면, 한 번에 크게 사는 순간 그 위 값까지 먹으면서 **내가 산 평균값이 밀려요.** 이걸 곧 직접 겪어볼 거예요.

**시각 지시:** 부록 A-1 호가 3열 구조. 최우선 매도호가의 잔량을 얇게 강조.

**미션:** (= 슬리피지 튜토리얼 A-3 전체) 최우선 매도호가가 150주뿐인 상황에서 **1,000주를 시장가로 사보세요.**

**피드백:** A-3 3~4단계 — 표시가 52,300 → 평균 체결가 52,460, 슬리피지 +160원(0.31%) 공개 → "화면 가격은 한 주의 값일 뿐, 크게 사면 위 호가까지 먹어요. 이게 슬리피지예요." + 차별화 펀치라인.

### 레슨 3 — 급등주를 쫓으면 생기는 일
- **소요:** ~2분 + 시나리오 · **해금:** 복기 리포트 · **태그:** `급등 추격` / `눌림목` 첫 등장

**개념 (미니 레슨 본문):**
> 빨간 급등을 보면 사고 싶어져요. 남들 다 버는데 나만 놓치는 것 같거든요. 이 조급함엔 이름이 있어요 — **FOMO**(놓칠까 봐 두려운 마음).
>
> 그런데 갑자기 뛴 가격은 종종 **원래 자리로 되돌아와요(평균회귀).** 늦게 올라탄 사람이 꼭지를 뒤집어쓰는 이유예요.
>
> 백문이 불여일견. 이번엔 **안전하게 한 번 데여봐요.** 진짜 돈이 아니니까요.

**시각 지시:** 급등 시나리오 진입 카드. 결정 지점에서 `지금 사기 / 첫 눌림까지 기다리기` 2선택 → 각각 태그 기록.

**미션:** 한빛전자가 급등하는 시나리오(→ Part 2 스펙 3). 오버슛 구간에서 **지금 살지, 첫 눌림까지 기다릴지** 고른다.

**피드백 분기 (시나리오가 평균회귀로 흐른 뒤):**
- 추격 매수(`급등 추격`) → 복기 미니 "고점 부근에서 샀어요. 방금 그 조급함이 FOMO예요. 다음엔 첫 눌림을 기다려볼까요?" → **복기 리포트 해금**.
- 기다림(`눌림목`) → "잘 참았어요. 조금 뒤 눌렸을 때 더 싸게 탈 수 있었죠. 기다림도 매매예요." → **복기 리포트 해금**.

> 레슨 3 완료 시점부터 매 매매에 태그가 붙고(부록 A-4), 주간 복기 리포트가 열린다. 즉 P0의 학습 루프가 여기서 닫힌다: 캔들로 차트를 읽고 → 호가로 체결을 이해하고 → 추격의 결말을 겪고 → 복기로 내 습관을 본다.

## Part 2. P0 기능 명세

### 스펙 1 — 배속(TimeScale) 〔부록 A-1 배속 칩〕
커리큘럼의 2번째 레슨부터 실시간 분봉과 충돌하므로(이평선 20캔들·터틀 20일) 시간 압축은 P0 엔진 요구사항이다(§2.2).

- **배율:** `1x`(실시간, 1틱 ≈ 1초) / `5x` / `30x`. 그 이상은 이산 스킵 버튼 `다음 캔들로` · `장 마감까지`.
- **구현:** 엔진 틱 간격 = 기준간격 ÷ 배율. 매칭·집계 로직은 불변, 틱 주기만 단축. 30x면 캔들이 초 단위로 완성된다.
- **압축 모드 프레이밍:** 시나리오·장기 레슨에선 "1캔들 = 1일" 라벨을 허용. 진입 시 "지금은 하루가 N초예요" 1회 고지, 시간축 라벨을 일 단위로 전환. 엔진은 동일, 라벨만 변경 — 교육적으로도 캔들의 스케일 불변성을 가르치는 포인트.
- **주문 정밀도 보호:** 배속 30x에서 주문 모달을 열면 자동으로 1x로 낮췄다가 닫으면 복귀(정밀 매매 방해 방지).
- **백그라운드 복귀 공유(P1):** 앱 재진입 시 경과 실시간을 동일한 catch-up 루프로 밀어 따라잡는다. `scenePhase` 감지. 배속과 메커니즘을 공유하므로 배속을 P0에 먼저 넣으면 P1 비용이 준다.

### 스펙 2 — 매매 사유 태그(TradeReasonTag) 〔§2.3〕
텍스트 입력이 아니라 **칩 1탭**. 마찰 최소화 + 구조화 데이터 확보(L1 복기·LLM 비용 절감).

- **진입(매수) 태그:** `돌파 기대` · `눌림목` · `급등 추격` · `뉴스·테마` · `그냥 감`
- **청산(매도) 태그:** `목표 도달` · `손절 원칙` · `무서워서` · `지루해서` · `그냥 감`
- **규칙:** 매매당 태그 1개 필수(1탭, ~0.5초), 텍스트 메모는 선택. `그냥 감`은 중립 스타일로 — 솔직한 기록이 데이터 품질의 전제이고, '감 매매의 성적표'가 가장 강력한 교육 콘텐츠가 된다.
- **데이터 모델(SwiftData):**
```
TradeLog {
  id: UUID
  timestamp: Date
  side: .buy | .sell
  symbol: String
  price: Decimal            // 주문 시 표시가
  qty: Int
  avgFillPrice: Decimal     // 실제 평균 체결가(슬리피지 반영)
  slippage: Decimal         // avgFillPrice - price
  reasonTag: TradeReasonTag
  note: String?             // 선택
  scenarioId: String?       // 시나리오 매매면
  seasonId: Int             // 계좌 시즌(리셋 단위)
  realizedReturn: Decimal?  // 청산 시 확정 수익률
}
```
- **L1 복기 쿼리:** `GROUP BY reasonTag → count, avg(realizedReturn)` = 부록 A-4 습관 분석의 원료. API 없이 오프라인·무료.

### 스펙 3 — 실패 시나리오 1종: 급등주 추격매수 (fairValue 경로) 〔§2.1·레슨 3〕
시나리오 = **fairValue(t) 경로 + 에이전트 파라미터 오버라이드 + 결정 지점**. 기존 엔진의 `fairValue` 진화·에이전트 구조를 그대로 쓰되 결정론적 시드로 재현 가능하게 한다(복기·봇 비교와 공유).

**fairValue 5단계 경로:**
| 단계 | 구간 | fairValue 거동 | 에이전트 오버라이드 |
|---|---|---|---|
| 1 기저 | t0–t1 | ~50,000 횡보(±노이즈) | 기본 |
| 2 촉발 | t1 | 52,000→58,000 급상승(뉴스 이벤트 트리거) | 추세추종 봇 activity·공격성 ↑ |
| 3 과열 오버슛 | t1–t2 | 60,000+로 fairValue 위 오버슛 **(← 미션 결정 지점)** | 노이즈 amplitude ↑, MM 스프레드 ↑(변동성) |
| 4 평균회귀 | t2–t3 | ~52,000으로 감쇠 → 급락 | 가치투자 봇 고평가 인식 → 매도 |
| 5 안정화 | t3– | 52,000 부근 횡보 | 기본 |

- **결정 지점:** 단계 3 오버슛 중 "지금 살까? 첫 눌림까지 기다릴까?" → 태그(`급등 추격`/`눌림목`) 기록.
- **결과 분기:** 추격 매수 → 단계 4에서 손실 → 복기. 기다림 → 단계 4 눌림에서 매수 기회 또는 관망 → 상대적 우위 체감.
- **엔진 스펙:**
```
ScenarioPreset {
  id: String
  seed: UInt64                              // 결정론적 재현
  fairValueKeyframes: [(tick: Int, value: Decimal)]   // 사이 보간
  agentOverrides: [(phaseRange, AgentParams)]         // 단계별 봇 파라미터
  decisionPrompts: [(tick: Int, prompt, tags: [Tag])] // 결정 지점
  timeScaleLabel: "1캔들 = 1일"                        // 압축 프레이밍
}
```
- 스토어 스크린샷·입소문 문구("일부러 물려보는 앱")를 이 시나리오로 만든다.

## Part 3. KPI 이벤트 계측 〔§2.8〕
P0에 계측을 심어야 §2.8 가설을 측정할 수 있다. **로컬 우선·익명 집계만**(금융 앱 신뢰).

### 이벤트 목록 (P0)
| 이벤트 | 주요 속성 |
|---|---|
| `onboarding_start` | — |
| `onboarding_level_choice` | choice: 완전처음 \| 해봄 |
| `first_trade_placed` / `first_trade_filled` | side, tag |
| `lesson_start` / `lesson_complete` / `lesson_dropoff` | lessonId, step |
| `tool_unlocked` | level(1–max) |
| `slippage_tutorial_completed` | avgSlippage |
| `trade_placed` | side, tag, wasTagSkipped(bool) |
| `tag_selected` | tag |
| `review_report_opened` | type: weekly \| postTrade |
| `review_report_dwell` | ms |
| `account_reset` | seasonId, endEquity, autopsyViewed(bool) |
| `session_start` / `session_end` | durationMs |
| `day_open` | dayIndex(설치 후 N일) |

공통 속성: `seasonId`, `appVersion`, `deviceTier`(온디바이스 LLM 가용 여부 — L2 복기 분기, §2.4).

### 지표 ↔ 이벤트 매핑
| 가설(§2.8) | 지표 | 계산 |
|---|---|---|
| 온보딩이 두려움을 이긴다 | 설치→첫체결 완료율 | `first_trade_filled` / `onboarding_start` |
| 학습 루프 작동 | 레슨1 완주 / 레슨2 진입 | `lesson_complete(1)` / `lesson_start(1)` · `lesson_start(2)` / `lesson_complete(1)` |
| **복기를 실제로 본다(핵심)** | 복기 열람률 | `review_report_opened` / `first_trade_filled` |
| 사유 기록이 마찰 아님 | 태그 완료율 | 1 − (`wasTagSkipped=true` / `trade_placed`) |
| 다시 온다 | D1 / D7 리텐션 | `day_open(1)` / `day_open(7)` ÷ 설치 |
| 리셋이 학습됨 | 부검 확인율 | `autopsyViewed=true` / `account_reset` |

> 복기 열람률이 목표(50%)에 못 미치면 차별화 3번(AI 코치) 전체를 재검토한다 — P1(L2·L3 복기) 투자 전에 P0에서 이 숫자를 확인하는 것이 계측의 핵심 목적이다.
