## 1. 한 줄 정의
실제 돈을 잃지 않고, 진짜처럼 살아 움직이는 가상 시장에서 매매하며 "왜 그렇게 움직이는지"를 배우는 주식 입문 앱.

## 2. 문제와 타깃
타깃은 두 부류의 주린이다.
- **시작이 두려운 사람**: 관심은 있지만 실제 돈을 넣기 무섭고, 차트·호가창·용어가 막막한 입문자.
- **잃고 돌아온 사람**: 실전에서 손실을 본 뒤 "내가 뭘 잘못했는지" 이해하고 다시 시작하려는 초보.

공통 페인은 **이론과 실전 사이의 간극**. 책·유튜브로 캔들과 PER을 외워도, 호가창 앞에서는 왜 가격이 그렇게 움직이고 내 주문이 어떻게 체결되는지 감이 없다.

타깃에 대한 제품의 직접 응답 두 가지: ① **회원가입 없이 시작**(두려운 사람 — 로컬 저장이므로 가능, §7) ② **온보딩에서 "완전 처음 / 해본 적 있음" 분기**(돌아온 사람에게 기초 UI 잠금을 강요하지 않음, §10.1).

## 3. 아이디어 검증 요약
- **시장성**: 검증됨. 증권플러스 모의투자(두나무), 마루랩 등 모의투자·금융교육 앱이 이미 존재. 수요의 신호이자, 블루오션이 아니라는 뜻.
- **핵심 리스크**: 차별화. "모의투자 + 퀴즈"는 마루랩이 이미 한다. 단순 복제는 의미 없음.
- **결론**: 진입 가치 있음. 단, §4의 차별화 wedge가 제품의 중심이어야 하고, **wedge가 첫 출시물(P0)에 반드시 포함되어야 한다**(v0.2의 구조적 결함 — wedge 전부가 P1 이후였음 — 을 v0.3에서 수정).

## 4. 차별화 전략 (제품의 심장)
세 가지의 결합이 경쟁 앱과 우리를 가른다.

1. **살아있는 가상 시장 엔진** — 가격 곡선을 미리 그리지 않는다. 성격이 다른 가상 트레이더(마켓메이커·노이즈·추세추종·가치투자)들이 호가창에 주문을 던지고, 가격은 체결의 결과로 emergent하게 발생한다. 사용자 주문도 같은 호가창에서 실제로 체결되어 슬리피지까지 재현된다.
2. **실패 시나리오 체험** — 안전한 가상 데이터이므로 위험한 상황을 의도적으로 재현할 수 있다. 급등주 추격매수의 결말, 급등 후 폭락, 횡보 인내 등 "한 번 데여봐야 아는 것"을 손실 없이 겪게 한다.
3. **AI 복기 코치** — 매수·매도 시 "왜 이 결정을 했는가"를 태그로 기록하게 하고, 결과가 나오면 매매 습관을 복기·피드백한다. 종목 추천이 아니라 "당신은 꼭지에서 추격매수하는 패턴이 있다" 같은 **메타인지 코칭**(§11 준수).

### 4.1 차별화는 증명되어야 한다 (v0.3 신설 원칙)
살아있는 엔진은 입문자가 체감으로 구분하지 못한다 — 호가창이 진짜 에이전트 주문인지 그려진 곡선인지 알 방법이 없다. 내부적으로 아무리 정교해도 **증명 장치가 없으면 차별화가 아니다.** 증명 장치는 **슬리피지 튜토리얼**(§7.2): 온보딩에서 일부러 큰 주문을 내게 해 평균 체결가가 밀리는 것을 직접 겪게 한다. 미리 그려진 차트 앱에서는 원리적으로 불가능한 경험이므로, 이 5분이 "이 앱은 다르다"를 증명한다.

## 5. 핵심 기능과 우선순위

### P0 (MVP — 차별화 포함형으로 개편)
- 살아있는 시뮬레이션 마켓: 실시간 분봉 형성, 호가창, 체결 테이프
- 모의 매수/매도(시장가), 포트폴리오(보유·평단·평가손익·현금)
- **시간 배속·스킵** (1x/5x/30x + 캔들 스킵 — 커리큘럼과 분봉 실시간의 구조적 충돌 해소, §9.2)
- **온보딩: 무가입 시작 + 슬리피지 튜토리얼** (§7)
- **실패 시나리오 1종: 급등주 추격매수** (§6 레슨 3의 미션 겸용)
- **태그 기반 매매 사유 기록 + L1 룰베이스 복기** (§8)
- **도구 해금 Lv0~2** (§10.1)
- 레슨 3종: 캔들 / 호가·슬리피지 / 추격매수의 결말 (사슬 구조, §6)
- 가상 자금 리셋 — **계좌 부검 게이트 포함** (§8.3)
- KPI 이벤트 계측 (§13 — P0에서 심지 않으면 가설 검증 불가)

### P1 (차별화 심화 — MVP 직후)
- L2 온디바이스 복기 + L3 API 심층 복기 (AI 코치의 계층화, §8.2)
- 거장 봇 2종(가치투자 거장형/추세추종형) + "나 vs 거장 봇" 비교 (§15)
- 터틀 전략 모듈 1종 (§15)
- 실패 시나리오 확장 + **오늘의 장**(일일 시나리오) + 주간 복기 리포트 푸시 (§6.2)
- 도구 해금 Lv3~max (호가창 확장·체결 테이프·지표 오버레이)
- 백그라운드 복귀 (배속과 catch-up 메커니즘 공유로 비용 절감)
- 결제·엔타이틀먼트 레이어 (§12)

### P2 (확장)
- 퀀트 빌더(조건→행동 규칙 조립 + 백테스트) — 출력은 순위·확률로만(§11.1·§15)
- 거장 봇 아키타입 확장
- 다종목·섹터, 지정가 주문, 공매도
- 실제 데이터 학습 모드(EOD/지연 시세 기반 과거 복기)
- **크립토 학습 모드(§16)** — 합성 기술적(무료) / 실 온체인(구독)
- 학습 진도·성취 트래킹 고도화, 랭킹·챌린지 등 가벼운 소셜

**솔로 개발 부하:** P0 추가분의 구현 비용은 배속(낮음 — 틱 간격 조절), 시나리오 1종(중), 태그+L1 복기(중 — SwiftData 집계), 해금(낮음 — 조건부 노출), 온보딩(중). LLM 복기를 P1로 이연한 것과 상쇄되어 총량은 크게 늘지 않는다.

## 6. 학습 루프 설계

### 6.1 세션 내부 루프
```
미니 레슨(1~2분) → 관련 모의매매 미션 → 즉시 피드백 + 복기 → 도구 해금
```
- **P0 레슨 사슬(3종):** ① 캔들(→ 캔들 차트 해금) → ② 호가·슬리피지(미션 = 슬리피지 튜토리얼, → 호가창 해금) → ③ 급등주 추격의 결말(미션 = 실패 시나리오, 태그 첫 등장, → 복기 리포트 해금). **각 레슨이 다음 도구를 연다** — 해금이 곧 보상(§10.1). 레슨 3 완료 시점에 P0 학습 루프가 닫힌다: 차트를 읽고 → 체결을 이해하고 → 추격의 결말을 겪고 → 내 습관을 본다.
- 실제 카피·미션·피드백 분기는 짝 문서 Part 1에 확정되어 있음.
- 이후 커리큘럼: 지지·저항/이평선 → 거래량 → 기본 지표(PER·PBR·ROE) → 심리·매매습관. 모든 레슨은 "읽기"로 끝나지 않고 시뮬레이션 안에서 행동으로 검증된다.
- **레슨별 시간 스케일 모드 명시**: 호가·체결 감각 = 분봉 실시간, 이평선·터틀·가치투자 = 압축 모드("1캔들=1일" 프레이밍 허용, §9.2). 이것이 v0.2 §11 "거장 봇 분봉 충돌" 리스크의 근본 해법이다.
- **세션 길이 설계:** 1세션 = 레슨 1 + 미션 1 + 복기 = 5~7분. 출퇴근 사용이 기본 시나리오.

### 6.2 리텐션 루프 (세션 바깥 — v0.3 신설)
- **오늘의 장** (P1): 매일 다른 시드·상황의 짧은 일일 시나리오. §12의 "무료 = 시나리오 일일 제한"과 맞물려 제한이 아니라 데일리 콘텐츠로 프레이밍된다.
- **주간 복기 리포트 푸시** (P1): 일요일 저녁 "이번 주 매매 12건, 당신의 패턴이 정리됐어요." 복기 코치가 리텐션 엔진을 겸한다.
- **절제된 스트릭**: 불꽃 배지가 아니라 학습 잔디(조용한 캘린더 히트맵). §10 톤 원칙과 정합.

## 7. 온보딩 (v0.3 신설)

### 7.1 첫 60초
1. **가입 없음.** 로컬 저장(SwiftData)이므로 가능 — "회원가입 없이 시작"을 화면에 명시.
2. "1,000만원이 준비됐어요. 진짜 돈은 1원도 안 듭니다." — 모의투자 고지(§11)를 안심 메시지로 겸용.
3. 분기 1문항: "완전 처음이에요 / 해본 적 있어요" → 시작 해금 레벨 결정.
4. **Lv0 화면**(§10.1): 종목 1개, 현재가, 라인 차트, 사기/팔기 버튼만. 잠긴 도구 카드가 흐릿하게 보인다.
5. 30초 안에 첫 매수 체결 → 축하가 아니라 관찰 유도: "가격이 계속 움직여요. 누가 움직이는 걸까요?"

### 7.2 슬리피지 튜토리얼 (엔진 존재 증명 — §4.1)
감정 곡선 4단계: **도발**(코치가 "겁내지 말고 크게 질러봐요" — 최우선 호가 150주뿐인데 1,000주 시장가 매수 유도) → **체결**(주문이 호가를 아래→위로 먹으며 체결 테이프에 찍힘) → **충격**(표시가 52,300 → 평균 체결가 52,460, 슬리피지 +160원 공개) → **깨달음**("화면 가격은 지금 살 수 있는 가장 싼 한 주의 값" + 차별화 펀치라인 "미리 그려진 차트 앱에선 절대 일어나지 않아요"). 이 튜토리얼이 레슨 2(호가·슬리피지)의 미션을 겸한다. 상세 스토리보드·산수 검증은 부록 A-3.

## 8. 복기 시스템 · 태그 · 시즌 (v0.3 신설 — 차별화 3번의 구현)

### 8.1 매매 사유 태그 (1탭 기록)
자유 텍스트 입력은 최악의 마찰 지점이므로 **칩 1탭 필수 + 텍스트 선택**으로 설계한다.
- 진입: `돌파 기대` `눌림목` `급등 추격` `뉴스·테마` `그냥 감` / 청산: `목표 도달` `손절 원칙` `무서워서` `지루해서` `그냥 감`
- 태그는 구조화 데이터라서 ① L1 룰베이스 복기가 즉시 가능 ② LLM 프롬프트 축소(API 비용 절감) ③ 태그별 성과 집계가 복기 리포트의 핵심 시각화가 된다.
- `그냥 감`은 부끄럽지 않게 중립 디자인 — 솔직한 기록이 데이터 품질의 전제. "감 매매의 성적표"가 가장 강력한 교육 콘텐츠다.

### 8.2 복기 3층 아키텍처 (비용 문제를 구조로 해결)
| 층 | 엔진 | 비용 | 제공 |
|---|---|---|---|
| L1 통계 복기 | 룰베이스 (태그×결과 집계) | 0원, 오프라인 | 무료 · 무제한 · **P0** |
| L2 요약 복기 | 온디바이스 LLM (Apple Foundation Models) | 0원 | 무료 · 지원 기기 한정 · P1 |
| L3 심층 복기 | Anthropic API | 유료 API | 구독 · P1~P2 |

- L1만으로 강력하다: "`급등 추격` 매수 4건 중 3건 손실, 평균 -3.2%."
- L2는 iOS 26+/Apple Intelligence 기기 한정 — **앱 타깃(iOS 18+)과 다르므로 가용 기기에서만 활성화, 미지원 기기는 L1 폴백**(§9).
- L3만 서버 비용이 발생하며 종단 패턴 분석·철학 렌즈 복기 등 온디바이스 불가능한 것만 담는다. → 무료/유료 경계가 인위적 제한이 아니라 **기술 계층의 자연스러운 경계**가 된다(§12).
- 코치 발화는 항상 사후·습관 관점(§11.1): "사라"가 아니라 "사는 속도가 빠를수록 성적이 나빴어요."

### 8.3 시즌과 계좌 부검 (리셋 설계)
무제한 원클릭 리셋은 손실의 무게를 지워 "가상이니까 몰빵" 행동을 강화한다 — 학습 목표와 정반대이자 모의투자 앱의 고질적 결함. v0.3 설계:
- **계좌 부검 게이트**: 리셋 버튼 → 이번 계좌의 사망 원인 리포트(사망 원인 헤드라인, 자산 곡선, 치명적 습관 3가지)를 스크롤한 뒤에야 리셋 가능. 리셋 자체를 학습 이벤트로 전환.
- **시즌 프레이밍**: 리셋된 계좌는 시즌 2, 시즌 3으로 기록. 시즌 간 성적 비교가 성장 그래프가 된다. **이월되는 것은 돈이 아니라 규칙**("한 종목에 30% 이상 넣지 않기").
- **파산을 콘텐츠로**: 계좌 -50% 도달 시 특별 레슨 해금 — "반토막의 수학: -51%면 원금 회복에 +106%가 필요하다."
- 화면 스펙은 부록 A-5.

## 10. 디자인 — 두 개의 시각 레지스터 (Claude Design 브리프, v0.3 전면 개정)

> v0.2의 "TOSS 스타일을 참고한 브랜드 톤 트레이딩 화면" 방향을 폐기하고, 실제 토스증권 화면 분석에 기반해 재정의한다.

앱을 두 개의 시각 언어로 분리한다.

| 레지스터 | 대상 화면 | 룩앤필 | 근거 |
|---|---|---|---|
| **시장(Market)** | 차트 · 호가 · 주문 · 포트폴리오 | **토스증권 충실 재현** — 흰 배경, 빨강 상승/파랑 하락, 볼드 숫자, 가격 사다리, 체결강도 | **전이 학습.** 여기서 익힌 감각이 실전 토스·키움에 1:1로 옮겨가야 한다. 낯선 UI는 학습을 리셋시키고, 친숙함은 두려운 타깃의 진입장벽을 없앤다. |
| **학습(Learning)** | 온보딩 · 레슨 · 복기 리포트 · 계좌 부검 · 코치 카드 | **아크 바이올렛** `#6B4EFF` 액센트 + 흰 배경/잉크 다크 — 베이지 톤 폐기(v0.3.1) | 경쟁 앱에 없는 우리만의 영역. 시장 화면과의 대비가 "학습이 시장 위에 얹혀 있다"를 전달한다. |

- **전환 자체가 메시지다**: 레슨(브랜드 톤) ↔ 실습(토스 톤)의 이동이 "이건 실전에서 볼 바로 그 시장"임을 말한다.
- **코치 카드 뒤집기**: 토스의 `✨ 왜 움직였을까?`(시장 설명)를 `✨ 왜 이렇게 샀을까?`(내 매매 설명)로 뒤집는다. 같은 카드 문법, 방향만 반대 — 토스 복제가 아니라 **토스 위의 학습층**임을 증명하는 한 장.
### 10.0 색 시스템 (v0.3.1 확정 — 베이지 폐기)

> Arcseed 초안의 bone/champagne 베이지 톤을 폐기한다. 토스·에어비앤비·틴더처럼 **흰 배경 + 강한 시그니처 컬러 하나**로 간다. 시그니처 후보 검증: 토스 블루는 우리의 하락색과 충돌+모방 리스크, 에어비앤비 코랄·틴더 핑크는 우리의 상승 빨강과 충돌. 한국 증권·핀테크에서 비어 있고 손익색과 절대 안 섞이는 슬롯은 **바이올렛**이다 — AI·학습·프리미엄의 연상까지 얹힌다.

**브랜드 — 아크 바이올렛 (Arc Violet)**
| 토큰 | 값 | 용도 |
|---|---|---|
| `violet500` (Primary) | `#6B4EFF` | 학습 CTA·코치 ✨·해금·프리미엄·모의 배지 |
| `violet600` (Deep) | `#4A32D9` | pressed 상태·다크 위 대비 강조 |
| `violet400` (On-dark) | `#9D8CFF` | 다크모드·잉크 카드 위 액센트 |
| `violet50` (Tint) | `#EFEBFF` | 코치 카드 배경·해금 칩 배경 |
| Gradient | `#6B4EFF → #9D5CFF` | 히어로·프리미엄 모먼트 한정(틴더식 절제 사용) |

**중립 (두 레지스터 공통)**
| 토큰 | 값 | 용도 |
|---|---|---|
| 배경 | `#FFFFFF` | 기본 캔버스(시장·학습 동일 — 베이지 대체) |
| 섹션 밴드 | `#F2F4F6` / 카드 `#F9FAFB` | 토스식 구분 |
| 본문 | `#191F28` / 보조 `#8B95A1` | 텍스트 |
| 다크 배경 | `#17171C` (잉크) | 다크모드·코치 카드 |

**기능색 (불변)**: 상승·매수 `#F04452` / 하락·매도 `#3182F6` / 잔량 틴트 매수 `#FEE8EA`·매도 `#EAF2FE` / 이평선 5 `#22C55E`·20 `#F04452`·60 `#F59E0B`·120 `#8B5CF6`.

두 레지스터의 구분은 이제 배경색이 아니라 **액센트의 존재**로 만든다: 시장 화면에는 바이올렛이 원칙적으로 등장하지 않고(코치 카드·모의 배지 예외), 학습 화면은 바이올렛이 주인공이다.

- **역할 분리 명문화**: 빨/파는 손익 전용(학습 화면에서도 손익·B/S 마커에만 차용). **바이올렛은 절대 손익에 쓰지 않고, 빨/파는 절대 브랜드에 쓰지 않는다.**
- **트레이드 드레스 안전선**: 관례는 따르되(빨/파·사다리·체결강도 — 업계 표준이라 소유 불가) 폰트·간격·강조색·모의 배지·코치 톤은 우리 것으로. "익숙하되 위조는 아닌" 선. 출시 전 법률 검토 항목에 포함(§11).

### 10.1 도구 해금 시스템 (정보 단계 노출의 확정 스펙)
| 레벨 | 해금 조건 | 화면에 추가되는 것 |
|---|---|---|
| Lv0 | 첫 실행 | 종목명 · 현재가 · **라인 차트** · 사기/팔기 |
| Lv1 | 캔들 레슨 | 캔들 차트 |
| Lv2 | 거래량 레슨 | 거래량 막대 · 이동평균선 |
| Lv3 | 호가 레슨 | 호가창 (4호가 → 이후 10호가) |
| Lv4 | 체결 레슨 | 체결 테이프 |
| Lv-max | 지표 레슨 | RSI·MACD 등 — **토스 밀도 전체 = 실전과 동일** |

- **해금이 곧 보상**: 게임화를 포인트·배지(도파민)가 아니라 "도구를 얻는" 성취로 치환.
- **Lv-max = 토스 밀도**: "토스보다 단순하게 시작 → 레슨을 거쳐 토스 밀도까지"가 곧 학습 곡선. 실제 토스 호가조차 초보에겐 압도적이라는 사실이 해금의 정당성이다.
- 잠긴 도구는 흐릿한 자물쇠 카드로 상시 노출("호가창 · 레슨 3에서 해금") — 다음 레슨의 동기.
- 숙련자는 설정 "모두 해금" + 온보딩 분기(§7.1)로 배려.

### 10.2 핵심 화면
① 시뮬 트레이딩(차트+호가+주문) ② 레슨/학습 루프 ③ **복기 리포트(시그니처 — 디자인 투자 1순위, 스토어 스크린샷 1번)** ④ 포트폴리오 ⑤ 계좌 부검. 8개 화면의 확정 스펙(구성·색값·카피 방향)은 **부록 A**에 고정되어 있다.

### 10.3 톤과 고지
- 톤: 차분하고 신뢰감 있게. 도파민 자극 절제, 학습 보상은 적당히.
- 색 규칙(한국식, 필수): 상승/매수 = 빨강, 하락/매도 = 파랑. 크립토 모드의 색 처리는 §16.4.
- 고지의 디자인 처리: `모의투자 · 실제 돈이 아닙니다` 배지를 바이올렛 아웃라인으로 상시 — 부끄러운 면책이 아니라 제품 정체성으로. 전략·봇 화면 하단 `교육용 전략 · 수익 보장 아님` 고정.

## 11. 데이터·법적 가드레일 (반드시 준수)
- **합성 데이터 우선**으로 라이선스 리스크 회피. 실시간 거래소 시세 재배포는 비용·계약상 솔로 불가.
- **개인 맞춤 종목 추천·매매 시그널 금지.** 대가를 받고 특정 종목을 추천·자문하면 자본시장법상 투자자문업 등록 대상(미등록 시 형사처벌 위험, 2024년 규제 강화). 본 앱은 교육·시뮬레이션으로만 포지셔닝.
- 실제 자금·실계좌 연동 없음. 모의투자임을 화면에서 즉시 고지(§10.3).
- "확실히 오른다" 같은 단정·수익 보장 표현 배제.
- **출시 전 법률 검토 1회 권장** — 항목: ① 자본시장법(추천·자문 경계) ② 실명 투자자 스타일 사용(§15.4) ③ **토스증권 트레이드 드레스**(§10) ④ 크립토 포함 시 가상자산이용자보호법(§16.4).

### 11.1 분석·예측 출력 원칙 (퀀트/AI 코치 공통)
퀀트·팩터 분석의 출력을 **점 가격 예측으로 표시하지 않는다.**
- **기술적으로 틀린 표현이다.** 멀티팩터 모델의 출력은 유니버스 내 상대 순위(percentile)와 보유기간 기대 상대수익·초과 확률이지 "X원이 된다"가 아니다. 점 가격은 거짓 확신을 준다.
- **법적으로 위험하다.** 목표가·상승 예측 제시는 추천·자문 금지선에 닿는다.

노출 형태는 percentile 랭크(상위 12%), 기대 상대수익 신호, 확률 분포(3개월 보유 시 시장 대비 +수익 확률 60%), 팩터 강약 프로파일로 한정. AI 복기 코치도 사후·습관 관점으로만 말한다(§8.2). 이 원칙은 "예측은 확률이지 확정이 아니다"라는 직관을 심는 교육 효과도 있다.

## 12. 수익화
원칙: **넓이(breadth)가 아니라 깊이·실데이터·편의(depth)를 게이팅한다.** 자산군(주식/크립토)은 무료로 열어 획득·리텐션을 만들고, 유료화는 심화에 건다. **광고는 도입하지 않는다**(금융 앱 신뢰 훼손).

### 12.1 3층 경계
| 영역 | 무료 (Free) | 프리미엄 |
|---|---|---|
| 학습 콘텐츠 | 기초 커리큘럼 전체, 입문 전략 1~2개 | 심화 전략 모듈(퀀트 멀티팩터, 가치 아키타입, 풀 터틀) |
| 시뮬레이션 | 주식·크립토 합성 연습 모두 개방, 시나리오 일일 제한("오늘의 장"으로 프레이밍, §6.2) | 무제한 시나리오, 매월 갱신 챌린지 |
| AI 복기 코치 | **L1 무제한 + L2 일 3회** (§8.2 — 기술 계층이 곧 가격 경계) | **L3 심층·무제한 복기** |
| 분석 도구 | 기본 기술지표(RSI·MACD·이평·거래량) | 백테스트 엔진, 포트폴리오 분석(Sharpe·MDD), 팩터 percentile 랭킹 |
| 데이터 | 합성 데이터만 | 실시장 데이터(EOD/지연), 크립토 온체인(§16) |
| 편의 | 로컬 저장 | 클라우드 동기화 |

크립토를 통째로 잠그지 않는다. 잠그는 것은 깊이·실데이터·편의뿐. **아크 바이올렛 = 유료·학습 시그널**(§10.0) — 별도 "PRO" 배지 없이 유료 영역이 시각적으로 학습된다.

### 12.2 결제 방식 매핑 — 일시결제와 구독 동시 운영
**비소모성 IAP (한 번 사면 끝나는 고정 콘텐츠):** 심화 전략 모듈 개별 언락(퀀트/가치/풀 터틀) + 전략 번들. 구독으로 걸면 둘째 달부터 해지 사유가 된다.
**자동갱신 구독 (매달 재발생하는 가치):** 실데이터 레이어(데이터 비용·라이선스가 곧 반복 가치), 매월 갱신 시나리오·챌린지, 백테스트·포트폴리오 분석, L3 심층 복기(API 소비), 무제한 시나리오, 클라우드 동기화.
**왜 둘 다:** 한국 사용자층은 구독 피로가 크다. "한 번 사서 영구 소유" 집단은 IAP로, 파워유저는 구독으로 — 지불 의향이 다른 두 집단을 모두 잡는다.

### 12.3 전환 트리거
1. 기초를 떼고 연습에 재미가 붙은 뒤 → 심화 전략 모듈 (IAP)
2. 합성으로 충분히 연습한 뒤 → 진짜 시장 데이터로 검증하고 싶을 때 (구독)
3. 오늘의 장(일일 제한)에 막힐 때 → 무제한 (구독)

### 12.4 StoreKit 2 구조
- Non-Consumable: `strategy.quant` `strategy.value` `strategy.turtle` + `strategy.bundle`(중복 구매 방지 로직 필요).
- Auto-Renewable: `pro.monthly` `pro.yearly`(동일 구독 그룹, 연간 할인).
- 엔타이틀먼트는 `Transaction.currentEntitlements`로 판정, 단일 소유 객체(@Observable)에 집약해 화면들이 파생 구독. 게이팅 로직을 한 곳에 모은다. **데이터 기능은 독립 플래그로 분리** — §12.6 플랜B 대비.

### 12.5 가격 앵커 (검증 필요)
IAP 모듈 개별 ₩3,900~5,900, 번들 ₩9,900~14,900 / 구독 월 ₩4,900~7,900, 연 ₩39,000~49,000. 검증 방법: 출시 후 A/B 이전에 ① 스토어 경쟁가 조사(증권플러스·마루랩·해외 Invstr/Bloom) ② 베타 지불의향 인터뷰(Van Westendorp 4문항).

### 12.6 선결과제
- **데이터 재배포 라이선스**: 실시장 데이터 표시에 KRX 등과의 계약이 필요할 수 있다. 구독 기둥을 실데이터로 세우기 전에 재배포 권리부터 확인.
- **구독 플랜B를 설계 단계에서 준비**: 라이선스가 막히면 구독 가치를 "L3 심층 복기 + 무제한 시나리오 + 월간 챌린지" 중심으로 재구성할 수 있도록 엔타이틀먼트를 분리 설계(§12.4).
- **크립토 규제**: 실 온체인 모드는 가상자산이용자보호법 영역(§16.4).

## 13. KPI (v0.3 신설)
MVP가 검증할 가설별 지표. 계측 이벤트 정의는 짝 문서 Part 3.

| 가설 | 지표 | 초기 목표 |
|---|---|---|
| 온보딩이 두려움을 이긴다 | 설치 → 첫 체결 완료율 | 70%+ |
| 학습 루프가 작동한다 | 레슨 1 완주율 / 레슨 2 진입율 | 60% / 40% |

| 사유 기록이 마찰이 아니다 | 태그 입력 완료율 | 95%+ |
| 다시 온다 | D1 / D7 리텐션 | 35% / 15% |
| 리셋이 학습된다 | 계좌 부검 확인율 | — (관찰) |

**복기 열람률이 가장 중요하다** — 이 숫자가 낮으면 차별화 3번(AI 코치) 전체가 흔들리므로, P1(L2·L3) 투자 전에 P0에서 검증한다.

## 15. 전략 시스템 & 거장 봇 확장 (P1~P2)

### 15.1 설계 원칙
- **새 아키텍처가 아니다.** 거장 봇·전략 봇은 기존 `Agent` 프로토콜을 구현하는 새 참여자다.
- **`fairValue`가 곧 내재가치 역할.** 가치형은 fairValue 대비 쌀 때 사서 버티고, 추세형은 돌파·모멘텀에 반응.
- **시간지평(horizon)이 핵심 구분 변수.** 가치형=장기, 추세형=단기. 코드와 화면 모두에 명시하고, 장기 지평은 압축 모드(§9.2)에서 체험시킨다.
- **두 축으로 노출**: (A) 전략 플레이그라운드(사용자가 규칙대로 매매) (B) 거장 봇(가상 참여자로 함께 두고 비교).
- **아키타입 라벨 우선**: "가치투자 거장형 / 추세추종형". 실명은 공개된 "스타일" 교육에 한정.

### 15.2 코드 스펙
**선행: 엔진 롤링 통계** — `MarketContext`에 추가: `highN`/`lowN`(돈치안), `atr`(변동폭·사이징·손절 기준), `maShort`/`maLong`.

```
TurtleAgent (추세추종 / horizon = .short)
  진입: close > highN(20) → 매수
  사이징: 1유닛 ≈ (계좌 1% 리스크)/(atr × 배수)  // 초기엔 고정 유닛 가능
  손절: 진입가 − 2×atr / 청산: close < lowN(10)
  피라미딩: 0.5×atr 유리할 때마다 +1유닛, 최대 4유닛

ValueInvestorAgent (가치투자 거장형 / horizon = .long)
  매수: lastPrice ≤ fairValue × (1 − marginOfSafety)  // 예 0.15
  보유: 장기, activity 낮게 / 매도: lastPrice ≥ fairValue × (1 + premium) 또는 fairValue 급락

MomentumQuantAgent (P2 / horizon = .short)
  매수: 골든크로스 / 청산: 데드크로스
```

**퀀트 빌더 (P2):** `Rule { indicator, comparator, threshold, action }` + `StrategyRunner`(백테스트→성과→시뮬 적용).
- 백테스트 지표: 총수익률, 샤프, MDD, 승률, 회전율, 거래횟수. **수익률 단독 노출 금지.**
- 출력 표현(§11.1 준수): percentile 랭크·기대 상대수익·확률만. 파이프라인: 팩터 → winsorize → 횡단면 z-score → 가중합 → 랭킹.
- **합성 데이터 제약**: 합성 캔들에선 기술적 팩터만 나온다. 가치·퀄리티 팩터(PER·PBR·ROE)는 가상 종목별 **합성 재무제표** 생성이 선행되어야 한다 — 이 분기를 모듈 착수 전에 확정.

**봇 비교 산출물:** 동일 시드·시나리오(ScenarioPreset 재현, §9.2)로 각 봇 자동 매매 → `BotComparison`(수익률, 거래 횟수, MDD, 매매 이유 타임라인).

### 15.3 디자인 스펙
- 터틀 모듈: 돈치안 채널 오버레이 + 진입/손절/청산 마커. 가이드 매매 후 "터틀 봇이라면" 비교.
- 퀀트 빌더: 조건→행동 블록 조립 UI. 백테스트 결과(수익 곡선·승률·MDD) 시각화, 종목 판단은 순위·확률 배지(점 가격 금지).
- 봇 아이덴티티 카드: 아키타입 라벨 + 한 줄 철학 + **시간지평 배지(장기/단기)** + 대표 규칙.
- "나 vs 거장 봇": 같은 시나리오 나란히 — 수치 + 매매 이유 타임라인. 코치 피드백을 선택한 철학 렌즈로(L3).
- `교육용 전략 · 수익 보장 아님` 상시 표시. 색은 §10 규칙.

### 15.4 콘텐츠·가드레일
- 시간지평 명시: 가치형 거장은 압축/시나리오 모드에서 의미 있고 분봉 스캘핑엔 부적합함을 솔직히 보여준다.
- **백테스트 우수 ≠ 실전 수익**: 유동성·뉴스·갭이 없는 시뮬레이션의 한계 고지.
- 실명 법무: 공개 철학의 "스타일" 교육은 가능하나 가짜 어록·추천·보증 금지. 안전 대안은 아키타입 라벨. 법률 검토 항목(§11).

## 16. 크립토 학습 모드 (P2)

### 16.1 포지셔닝
크립토를 **자산군 다양성**으로 무료 개방하되, 크립토 고유의 심화 분석(온체인)은 구독에 둔다. 통째로 잠그면 차별화가 약하고 결제 동기가 안 나온다.

### 16.2 무료 = 합성 기술적
- 기존 엔진 4모듈 **그대로 재사용** — 크립토는 심볼·변동성 파라미터가 다른 시장일 뿐(24시간·고변동성 파라미터 조정).
- 지표는 주식과 동일한 기술적 지표뿐. 실패 시나리오·복기 루프 동일 적용.

### 16.3 구독 = 실 온체인
- 크립토의 "펀더멘털" = 온체인 데이터: MVRV·MVRV Z-score, NVT, SOPR, 실현가격, 활성 주소·해시레이트, 거래소 netflow, 보유자 분포, 펀딩비, OI, 공포·탐욕 지수.
- **온체인은 합성으로 만들 수 없다** — 실데이터 API(글래스노드류) 월 비용이 곧 구독의 반복 가치. 표현은 §11.1 준수.

### 16.4 규제·색 규칙
- 규제: 가상자산이용자보호법 영역. "모의·교육이며 투자 신호가 아니다"를 크립토 진입·유료 진입 화면에 명시. 법률 검토 항목(§11).
- 색: 앱 전체 한국식(상승 빨강/하락 파랑) 유지. 글로벌 거래소는 반대(상승 초록)인 경우가 많으므로 **차이를 교육 포인트로 명시** — 진입 시 1회 카드 고지.

### 16.5 대상 사용자 주의
주린이가 크립토에서 막히는 지점은 온체인 퀀트가 아니라 **개념과 리스크**(변동성·지갑·거래소·24시간 시장)일 가능성이 높다. 무료 크립토 콘텐츠의 중심은 개념·리스크 교육에 두고, 온체인 퀀트는 파워유저용 구독 심화로 분리한다.
